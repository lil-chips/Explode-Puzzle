# crypto-analysis.md — Crypto Investment Analysis Playbook

This playbook defines how to analyze cryptocurrency and blockchain projects.

The goal is to separate real value from speculation.

Crypto markets contain large amounts of hype, narrative trading, and misinformation.  
This framework prioritizes structure, incentives, and risk awareness.

---

# Core Objective

Determine:

1. what the protocol actually does
2. who uses it and why
3. whether the token is necessary
4. how value accrues to the token
5. what risks could destroy the thesis

Avoid narrative-driven decisions.

Focus on incentives and real usage.

---

# Step 1 — Define the Protocol

Start with a simple explanation.

Questions:

- What is the protocol?
- What problem does it solve?
- Who is the target user?
- Is this infrastructure, application, or middleware?

Examples:

- Layer 1 blockchain
- Layer 2 scaling solution
- DeFi protocol
- oracle network
- storage network
- gaming ecosystem

Output:

- protocol category
- purpose
- target users

---

# Step 2 — Understand the System

Explain how the system works.

Questions:

- What happens when someone uses the protocol?
- What participants exist?

Typical actors:

- users
- developers
- validators
- liquidity providers
- governance participants

Output:

- system architecture
- participant roles
- activity flow

---

# Step 3 — Token Utility

One of the most important questions:

**Does the token actually have a purpose?**

Questions:

- Is the token required to use the protocol?
- Is it used for fees?
- Is it used for governance?
- Is it used for staking or security?
- Does demand for the protocol increase demand for the token?

Danger signals:

- token exists only for speculation
- token utility is weak or artificial

Output:

- token purpose
- real vs artificial demand

---

# Step 4 — Tokenomics

Understand how tokens are distributed and issued.

Key areas:

### Supply

- max supply
- circulating supply
- inflation rate

### Distribution

- team allocation
- investor allocation
- community allocation
- ecosystem incentives

### Unlock schedule

Questions:

- Are large unlocks coming?
- Will investors dump?
- Is dilution high?

Output:

- token supply structure
- inflation risk
- unlock risk

---

# Step 5 — Incentive Design

Crypto systems depend on incentives.

Questions:

- Are incentives aligned?
- Are validators rewarded properly?
- Are users rewarded for real activity or fake farming?
- Does the system require constant subsidies?

Red flags:

- unsustainable rewards
- yield without real revenue
- short-term incentive farming

Output:

- incentive alignment
- sustainability

---

# Step 6 — Adoption and Usage

Look for **real usage**.

Metrics may include:

- active users
- transaction count
- transaction fees
- TVL (total value locked)
- developer activity
- ecosystem growth

Questions:

- Are people actually using the protocol?
- Are users sticky?
- Is usage growing or declining?

Output:

- adoption metrics
- growth trend

---

# Step 7 — Competitive Landscape

Crypto is extremely competitive.

Questions:

- What other protocols solve the same problem?
- What advantage does this protocol have?
- Can competitors easily replace it?

Possible advantages:

- network effects
- developer ecosystem
- liquidity dominance
- security reputation
- infrastructure integration

Output:

- competitive strengths
- weaknesses
- risk of replacement

---

# Step 8 — Security

Security failures can destroy value instantly.

Check:

- smart contract audits
- past exploits
- bridge security
- validator decentralization
- governance attack risk

Questions:

- Has the protocol been audited?
- Has it been hacked before?
- Are funds safe?

Output:

- security quality
- major vulnerabilities

---

# Step 9 — Governance

Crypto governance affects long-term stability.

Questions:

- Who controls governance?
- Can insiders dominate votes?
- Are decisions transparent?
- Is governance active or stagnant?

Red flags:

- governance capture
- centralized control
- inactive community

Output:

- governance structure
- decentralization level

---

# Step 10 — Catalysts

Events that may influence price or adoption.

Examples:

- major partnerships
- exchange listings
- protocol upgrades
- ecosystem launches
- token unlock completion
- new incentive programs

Output:

- short-term catalysts
- long-term catalysts

---

# Step 11 — Risk Analysis

Identify major risks.

Categories:

- smart contract risk
- regulatory risk
- token dilution
- governance attacks
- ecosystem dependency
- loss of developer interest
- liquidity collapse

Questions:

- What could break the protocol?
- What could collapse token value?

Output:

- key risks
- severity
- likelihood

---

# Step 12 — Valuation Perspective

Crypto valuation is difficult but still necessary.

Possible approaches:

- market cap comparison
- fully diluted valuation (FDV)
- revenue multiples (if protocol generates fees)
- token velocity
- adoption potential

Questions:

- Is the token overvalued relative to usage?
- Is valuation justified by growth potential?

Output:

- valuation assessment
- overvalued / fair / undervalued perspective

---

# Final Investment View

Summarize the analysis.

Answer:

- What is the investment thesis?
- What must happen for this to succeed?
- What risks could invalidate the thesis?
- Is this a high-risk speculation or strong long-term bet?

Possible conclusions:

- attractive long-term opportunity
- speculative opportunity
- watchlist
- too risky
- fundamentally weak

---

# Red Flags

Extreme caution if you see:

- unclear token utility
- excessive token inflation
- large insider allocations
- weak security practices
- fake usage metrics
- unsustainable yield incentives
- heavy reliance on hype

---

# Final Principle

Crypto investing requires extreme skepticism.

Many projects exist only because speculation exists.

Focus on:

- real usage
- sustainable incentives
- security
- strong developer ecosystems

Ignore noise and narratives.
