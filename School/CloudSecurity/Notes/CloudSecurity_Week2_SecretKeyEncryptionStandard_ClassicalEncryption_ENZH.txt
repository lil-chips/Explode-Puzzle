1) Model of Conventional Encryption (the standard diagram)
====================================================================

1.1 Concept
EN: Conventional (secret-key) encryption uses the same secret key for encryption and decryption (or keys that are trivially derived from each other).
ZH: 传统/对称加密使用同一把密钥进行加密和解密（或两把可轻易相互推导的密钥）。

EN: The model includes: plaintext P, encryption algorithm E, secret key K, ciphertext C, decryption algorithm D, and an adversary who may observe C and try to recover P or K.
ZH: 模型包含：明文 P、加密算法 E、密钥 K、密文 C、解密算法 D，以及能观察密文并尝试恢复明文或密钥的攻击者。

1.2 Intuition
EN: The algorithm can be public; security should rely on the secrecy of the key, not the secrecy of the algorithm (Kerckhoffs’ principle).
ZH: 算法可以公开；安全性应依赖密钥保密，而不是依赖算法保密（Kerckhoffs 原则）。

1.3 Exam traps
EN: If a question says “the adversary knows the algorithm,” that is normal — do not assume security-by-obscurity.
ZH: 题目说「攻击者知道算法」是正常设定，不要假设靠隐藏算法来安全。

2) Substitution cipher (and Caesar as earliest example)
====================================================================

2.1 Concept
EN: Substitution cipher replaces each plaintext symbol with another symbol (letter/number/symbol). In simple monoalphabetic substitution, each plaintext letter maps to exactly one ciphertext letter.
ZH: 替换密码把每个明文字母替换成另一个符号。简单的单表替换是：每个明文字母固定映射到一个密文字母。

2.2 Intuition
EN: You can think of it as a secret “dictionary” mapping between alphabets.
ZH: 可以把它理解为一份秘密「字母字典/对照表」。

2.3 Example: Caesar cipher
EN: Caesar cipher shifts letters by 3 positions. Example shown:
Plain:  PLEASE CONFIRM RECEIPT
Cipher: SOHDVE FRQILUP UHFHLSW
Key k=3.
ZH: 凯撒密码把字母整体向后移 3 位。示例：
明文 PLEASE CONFIRM RECEIPT → 密文 SOHDVE FRQILUP UHFHLSW，k=3。

2.4 Common mistakes
EN: Confusing Caesar cipher with general shift cipher. Caesar is just one fixed key (k=3); shift cipher allows any k.
ZH: 把凯撒密码与一般移位密码混淆。凯撒固定 k=3；移位密码允许任意 k。

3) Shift cipher (formal math representation)
====================================================================

3.1 Formal definition (cryptosystem as a 5-tuple)
EN: A cryptosystem is a 5-tuple (P, C, K, E, D):
- P: set of possible plaintexts
- C: set of possible ciphertexts
- K: keyspace (set of keys)
- E: set of encryption functions {e_k}
- D: set of decryption functions {d_k}
such that for every k and every plaintext x: d_k(e_k(x)) = x.
ZH: 密码系统可形式化为五元组 (P, C, K, E, D)：
- P：明文集合
- C：密文集合
- K：密钥空间
- E：加密函数集合
- D：解密函数集合
并满足：对任意 k 与任意 x，有 d_k(e_k(x))=x。

3.2 Mapping letters to numbers
EN: Use Z26 = {0,1,2,...,25}. Often A=0, B=1, ..., Z=25.
ZH: 使用 Z26 = {0..25}；通常 A=0, B=1,..., Z=25。

3.3 Encryption/Decryption
EN: Shift cipher:
- Keyspace: K = {0,1,...,25}
- Encrypt: c = p + k (mod 26)
- Decrypt: p = c − k (mod 26)
ZH: 移位密码：
- 密钥：K={0..25}
- 加密：c=p+k (mod 26)
- 解密：p=c−k (mod 26)

3.4 Modular arithmetic mini-toolkit
EN: Definition: x (mod n) is the remainder after dividing x by n.
Example: 34 (mod 26) = 8.
ZH: 定义：x (mod n) 是 x 除以 n 的余数。
例：34 mod 26 = 8。

EN: Handling negatives (important!):
If you get −2 (mod 26), add 26 → 24.
ZH: 处理负数：
−2 mod 26 可加 26 变成 24。

3.5 Worked example (from slide)
EN: p='y'=24, k=3 → c=27 mod 26=1='b'.
Decrypt: c='b'=1 → p=1−3=−2 mod 26=24='y'.
ZH: 例：y(24) +3 → b(1)；解密 1−3=−2 mod 26=24。

3.6 Security discussion
EN: Without the key, decryption is easy: try all 26 shifts or use frequency + recognisable words.
ZH: 不知道密钥也很容易解：试 26 种移位或用词频/可读性判断。

6) Exam traps & common mistakes (Week 2)
====================================================================

EN: Trap 1 — Confusing “remainder” with “mod class”. In crypto, we treat values modulo n; you can add/subtract multiples of n freely.
ZH: 陷阱1——把「余数」与「模等价类」混在一起。模 n 下可以自由加减 n 的倍数。

EN: Trap 2 — Forgetting that decryption must be a function that inverts encryption for every plaintext.
ZH: 陷阱2——忘了解密必须对所有明文都能逆回去。

EN: Trap 3 — Saying “algorithm secrecy” provides security.
ZH: 陷阱3——认为隐藏算法就安全。
